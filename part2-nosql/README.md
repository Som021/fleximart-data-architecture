Part 2 — NoSQL (MongoDB)

 Objective:
This part demonstrates the use of MongoDB for handling flexible, schema-less product catalog data.

---

 Components

Files Included:
- `nosql_analysis.md` — NoSQL vs SQL comparison
- `products_catalog.json` — Nested product catalog
- `mongodb_operations.js` — MongoDB CRUD & aggregation queries

---

 Why MongoDB?:
- Flexible schema for product variants
- Faster iteration for evolving data
- Natural JSON document structure
- Suitable for catalog and inventory systems

---

Key Features:
- Nested categories and variants
- Stock tracking
- Aggregation pipelines
- Index usage

---

 How to Run:
On Bash: 

mongo < mongodb_operations.js
